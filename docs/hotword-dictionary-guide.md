# 热词字典功能说明

热词字典用于提升语音输入后的文本准确度。用户可以把容易识别错误的词配置成“识别词 -> 标准词”，后端整理文本时会自动替换。

## 功能范围

当前热词字典支持：

- 查看内置热词。
- 添加自定义热词。
- 删除自定义热词。
- 防止重复添加已经存在的识别词。
- 文本整理时实时读取最新热词。

内置热词随代码版本管理，自定义热词保存在本地数据文件中。

## 后端接口

### 获取热词列表

```http
GET /api/hotwords
```

### 添加热词

```http
POST /api/hotwords
Content-Type: application/json

{
  "source": "小七",
  "target": "七牛云助手"
}
```

### 删除自定义热词

```http
DELETE /api/hotwords/{source}
```

内置热词不能通过该接口删除，只能通过代码配置维护。

## 存储方式

默认内置热词文件：

```text
backend/app/config/text_rules.json
```

用户自定义热词文件：

```text
backend/data/hotwords.json
```

`backend/data/hotwords.json` 是本地运行数据，已经被 `.gitignore` 忽略，不会提交到代码仓库。

## 使用建议

适合添加为热词的内容包括：

- 人名、团队名、项目名。
- 课程名、专业名、论文关键词。
- 产品名、技术名词、英文框架名。
- 语音识别经常误写的同音词。

例如：

```text
小七 -> 七牛云助手
派森 -> Python
法斯特 api -> FastAPI
```

## 工程取舍

热词字典是低成本提升准确度的方式，不依赖在线语音 API，也不需要本地模型。它适合作为 MVP 阶段的准确度增强能力。后续接入在线语音识别 API 后，热词字典仍然可以作为识别结果的后处理规则继续使用。
