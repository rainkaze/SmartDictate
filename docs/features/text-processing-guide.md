# 文本处理规则说明

SmartDictate 的文本处理能力集中在后端 `TextProcessor` 中，主要负责把语音识别得到的原始文本整理成更适合直接复制和使用的结果。

## 设计原则

- 规则配置和处理逻辑分离，避免把大量词表硬编码在 Python 类中。
- 文本处理结果必须可预测，方便写单元测试和评审复现。
- 默认规则保持轻量，不依赖外部服务，保证项目本地可运行。
- 后续可以在不破坏接口的前提下接入在线大模型或语音识别 API。

## 规则文件

规则文件位置：

```text
backend/app/config/text_rules.json
```

当前包含三类配置：

- `filler_words`：口癖词，例如“嗯”“呃”“那个”“就是”。
- `hotwords`：热词纠错，例如把“派森”替换为“Python”。
- `scene_prefixes`：场景前缀，例如会议纪要、学习笔记、代码说明。

## 处理流程

后端收到原始文本后，会按以下顺序处理：

1. 规整空白字符。
2. 删除口癖词。
3. 应用热词替换。
4. 合并语音识别中可能出现的词间空格。
5. 对“然后、但是、所以”等明显连接词做轻量停顿处理。
6. 合并重复标点。
6. 根据输入场景添加前缀。
7. 返回整理结果和指标信息。

## 指标说明

接口会返回这些指标：

- `raw_length`：原始文本长度。
- `processed_length`：整理后文本长度。
- `removed_fillers`：清理掉的口癖词数量。
- `estimated_reading_seconds`：估算阅读耗时。

## 后续扩展

后续可以增加：

- 用户自定义热词。
- 按场景配置不同的口癖词和标点策略。
- 接入在线大模型做智能润色。
- 接入在线语音识别 API，把音频直接转成文本。
