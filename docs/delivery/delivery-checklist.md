# 交付检查清单

提交作品前按此清单检查，避免因为文档、启动方式或仓库状态影响评审。

## 仓库

- [ ] GitHub 仓库可公开访问。
- [ ] Gitee 镜像同步正常。
- [ ] 主分支代码可运行。
- [ ] 没有提交 `.env`、本地历史数据、`node_modules`、`dist`。
- [ ] PR 描述清晰，包含功能描述、实现思路、测试方式和风险说明。
- [ ] Commit 分布不是最后一天一次性导入。

## 启动

- [ ] `scripts\start-backend.cmd` 可以启动后端。
- [ ] `scripts\start-frontend.cmd` 可以启动前端。
- [ ] `http://127.0.0.1:5173` 可以访问页面。
- [ ] `http://127.0.0.1:8000/docs` 可以访问接口文档。

## 测试

- [ ] `python -m pytest` 通过。
- [ ] `python -m ruff check backend` 通过。
- [ ] `cd frontend && npm.cmd run build` 通过。

## 功能

- [ ] 可以点击“填入示例”。
- [ ] 可以整理文本。
- [ ] 可以复制整理结果。
- [ ] 可以添加自定义热词。
- [ ] 自定义热词能在下一次整理时生效。
- [ ] 历史记录可以回填、复制、删除。
- [ ] 后端未启动时前端有明确提示。

## README

- [ ] 说明项目定位。
- [ ] 说明启动方式。
- [ ] 说明核心功能。
- [ ] 说明工程取舍。
- [ ] 放置 Demo 视频链接。
- [ ] 放置截图或 GIF。

## Demo 视频

- [ ] 有声音讲解。
- [ ] 展示语音输入或示例输入。
- [ ] 展示文本整理。
- [ ] 展示热词字典。
- [ ] 展示历史记录。
- [ ] 展示启动方式或 README。
- [ ] 视频链接放在 README 显眼位置。
