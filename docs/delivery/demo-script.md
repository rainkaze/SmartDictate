# Demo 演示脚本

本文档用于录制 Demo 视频或路演时参考。建议控制在 3 到 5 分钟，重点展示完整闭环和工程取舍。

## 1. 项目定位

SmartDictate 是一个本地优先的语音输入助手，目标是帮助用户更高效地完成文本输入。它不是系统级输入法驱动，而是一个可本地运行的 Web 语音输入产品，更适合在短周期内交付完整 MVP。

## 2. 启动项目

后端：

```bash
scripts\start-backend.cmd
```

前端：

```bash
scripts\start-frontend.cmd
```

访问：

```text
http://127.0.0.1:5173
```

## 3. 演示基础整理

如果环境不方便说话，点击“填入示例”。

示例文本：

```text
嗯 我想用派森开发七牛语音输入法，然后让小七帮我整理文档
```

点击“整理文本”，说明：

- “嗯”“然后然后”会被清理。
- “派森”会被纠正为 “Python”。
- 文本会进行轻量标点整理，不会把每个空格都机械替换成逗号。
- 结果会保存到最近记录。

## 4. 演示热词字典

在“热词字典”中添加：

```text
识别词：小七
标准词：七牛云助手
```

再次整理包含“小七”的文本，展示热词即时生效。

说明热词字典的价值：

- 适配人名、课程名、项目名、技术词。
- 不依赖在线 API，成本低。
- 后续接入在线语音识别后仍可作为后处理规则。

## 5. 演示历史记录

展示最近记录：

- 填入：把历史结果放回编辑区。
- 复制：快速复制历史结果。
- 删除：清理单条记录。
- 清空：清空全部记录，便于演示重置。

## 6. 讲清楚工程取舍

建议说明：

- 当前识别使用浏览器 Web Speech API，成本低、启动快。
- 后端负责文本整理、热词和历史记录，保证产品闭环。
- 自定义热词和历史记录保存在本地 SQLite 数据库中，不提交仓库。
- 后续可以扩展在线语音识别 API 或本地 Whisper 模式。

## 7. 结尾总结

SmartDictate 的 MVP 已经完成：

```text
语音输入 -> 文本整理 -> 热词纠错 -> 历史复用
```

这条链路覆盖了题目要求中的准确度、易用性、响应速度和成本权衡。
